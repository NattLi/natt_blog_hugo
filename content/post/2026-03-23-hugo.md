头图·当前写blog的方式。  


#### WordPress

从差不多2009年开始，写点东西要更新在Bokee、校内网、MSN Space 等地方，慢慢想到，万一这些平台死了怎么办？于是，机缘巧合我接触到了WordPress，也折腾过一些虚拟主机，也是在那个时候，开始明白了网站的基本逻辑，并且把所有数据拿在手上的感觉很好。

慢慢的，学会了OSS做CDN和云备份。学会了博客搬家和换域名等操作。

这种自己整一个云主机搭一个环境折腾一个网站的感觉挺好。


#### Jekyll

2018年左右，又是机缘巧合，我接触到了github这个新世界，了解到了git工具的神奇和markdown的简洁。于是，我把自己工作中的一些文字工作交给了markdown，也尝试了在github上托管GitBook生成的网页。

这算是第一次接触静态网站生成器。 

也用到了一些命令行。慢慢的，我发现blogging也可以通过类似GitBook的方式进行。 

于是，我fork了某位朋友的blog，修改配置之后形成了自己的blog。这个技术底，是Jekyll, 托管的服务在 Github Pages，生成的服务，莫名其妙用的 TravisCI.

但，这个阶段，我始终觉得 Jekyll 虽然打字很爽，但始终不是类似WordPress这般像个正经玩意，在 IDE 里用 Markdown 进行 Blogging 虽然很爽，但总觉得少了点什么。WordPress 站的感觉才似乎是一个正经网站的样子。

这个阶段，Jekyll 和 WordPress 共存了几年。直到2026年3月。

#### Hugo

2026年1月，已经用了几个月AI驱动的IDE Antigravity, 可以在我仅有的那一点点互联网和程序基础上帮我迅速完成想要的目标。于是，在它的帮助下，我对比了 Jekyll, Hugo, Hexo 等静态网站生成器，最终选择了 Hugo。

并且，导出导入了之前 Jekyll 和 WordPress 的所有内容，并顺利导入到 Hugo. 并且，在 Antigravity 的帮助下，我把 CloudFlare 的 CDN 和 DNS 也迁移到了 Hugo 上。

当前，技术层面，我在IDE里写markdown，然后commit push 到 github，CloudFlare 自己生成静态网页然后部署进自己的CDN，整个过程非常丝滑。甚至节省了每年没几个钱的 VPS 直出。

更重要的是，github 背靠微软爸爸，短时间应该不会倒闭，而且所有的数据都有一套基于md的裸数据，甚至可以在IDE里脱机阅读，github就算倒了问题也不大。

所以，就有了这个新的blog。当前已经替换了域名，之前的WP站虽然没关闭，但是域名已经切换了，并且，我把WP站的网站文件和数据库做了备份下载，算是给那个时代一个脱水备份吧。
